# Phase 1 Reusable UI Patterns

## Forms
- labeled inputs with helper text
- inline validation messages
- grouped sections for related fields
- disabled and loading submit states

## Tables
- sortable columns where useful
- row click to detail
- mobile card fallback
- empty state and loading skeleton

## Filters
- branch filter
- date filter
- status filter
- search input with debounced query later in implementation

## Detail Views
- summary card
- status badge
- action bar
- history sections with timestamps

## Feedback Components
- success banner or toast
- inline error alert
- destructive action confirmation modal
- permission-denied state

## Status Presentation
- consistent status badge system for:
  - active
  - expired
  - inactive
  - suspended
  - frozen
  - pending payment
  - trial

## Search Pattern
- persistent search input on members pages
- manual lookup field on check-in
- result list with enough metadata to avoid wrong-member actions
